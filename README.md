# Sprint-Review-and-Retrospective

## Reflection

Understanding user needs starts with actually looking at what the user is asking for and breaking it down into something more specific. Instead of thinking about the program as a whole, I focus on smaller pieces of functionality. This is where user stories come in. User stories help translate what the user wants into clear, simple goals that can be worked on step by step. They also make it easier to prioritize features and understand why each part of the program is important.

When it comes to developing programs, I prefer taking an Agile approach because it allows for flexibility and continuous improvement. Instead of trying to build everything at once, I like breaking work into smaller sprints and making adjustments along the way. In the future, I want to keep using Agile practices like sprint planning, daily standups, and retrospectives because they help keep projects organized and make sure progress is actually being made. This approach also makes it easier to handle changes without completely restarting the project.

Being a good team member in software development means communicating clearly, staying accountable, and being willing to collaborate with others. Everyone on the team has a role, but the project only works if people are willing to share ideas and help each other when needed. Good teamwork also means being open to feedback and making improvements based on it. Overall, a strong team member helps keep the project moving forward while supporting the rest of the team.
